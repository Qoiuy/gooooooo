包名
当命名包时，请按下面规则选择一个名称：

全部小写。没有大写或下划线。
大多数使用命名导入的情况下，不需要重命名。
简短而简洁。请记住，在每个使用的地方都完整标识了该名称。
不用复数。例如net/url，而不是net/urls。
不要用“common”，“util”，“shared”或“lib”。这些是不好的，信息量不足的名称。
另请参阅 Package Names 和 Go 包样式指南.


## Package Names

包名小写 不加下划线和驼峰

缩写包的名称 会使其含糊不清的情况  不要这样做

站在用户的角度定义包名 包名和它的内容名是耦合的

包里面的struct 不需要起和包名相关的名字了 例如
http.Server  而不是 http.httpServer

简化函数名称 例如  
time.Now()  而不是 time.NowTime()

New在包中特殊返回
list.New()   而不是  list.NewList()

如果您不能为包的内容提供一个有意义的前缀的包名称，则包抽象边界可能是错误的。


包名也是可以重构的 !!!
